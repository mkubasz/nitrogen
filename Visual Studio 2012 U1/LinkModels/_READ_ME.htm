Link UML Elements to Diagrams in Visual Studio Ultimate

Distributed on MSDN Code Samples Gallery

UML elements such as use cases or classes can be linked to another element, UML diagram, or any other file by using this extension of Visual Studio Ultimate. Users create links by dragging, and can later open the link target by double-clicking.

When the user double-clicks a linked element, if the link target is:

  • A UML diagram in the same Visual Studio solution: the diagram will open and come to the front in Visual Studio.
  • A specific UML element: it will be selected.
  • A word or phrase in a Word doc: the document will open with the phrase selected.
  • A slide in PowerPoint.
  • Any other type of file: it will open in the appropriate application.

Links are stored as relative file paths, so that they will work for other team members if the target files are part of the same Visual Studio solution.

A small icon is displayed on items that have a link.

You can easily adapt the sample code to your own needs. For example, you could add rules that make the name or other attributes of the linked element reflect features of the linked element. The UML tools in Visual Studio Ultimate are implemented on Visualization and Modeling SDK.

The source code demonstrates a number of techniques:

  • How to associate data with any UML element by using References.
  • How to add decorators to UML elements
  • How to add rules to respond to changes in a UML model
  • How to define menu commands for UML
  • How to handle the user dragging various kinds of object onto a UML diagram:
    • Project Items from Solution Explorer
    • Files from Windows Explorer
    • Bookmarks from Word
    • Shapes from other UML diagrams

How to run the sample

  1. You need:
    • Visual Studio 2010 Ultimate (or SP1)
    • Visual Studio SDK
    • Visualization and Modeling SDK ( = DSL Tools)
  2. Download, unzip, open UmlElementLink.sln
  3. Press F5 to build and run the sample.
    • The sample includes a test model, which will open in an experimental instance of Visual Studio.
  4. In any diagram in the test model, drag any of the following items onto any shape on the diagram:
    • Any file from Solution Explorer, in the same project or another project.
    • Any file from Windows Explorer.
      • Note that the link will stop working if you later move the solution and the target file into different relative locations. If you use source control, this means that it is advisable to link to files that are in source control; and even better to link to files in the same Visual Studio solution.
    • Any element from another UML diagram, in the same UML model (project) or another one. To do this, either display the diagrams side by side; or drag the element up to the tabs, across to the tab of the diagram where you want to drop, and then down to the shape you want to drop onto.

How to install the sample on other computers for demonstration purposes

To install the extension on other computers:

  1. The target computer must have Visual Studio Ultimate. It does not need VSSDK or VMSDK.
  2. Copy the VSIX file from bin\Debug to the target computer.
  3. Double-click the VSIX.

To uninstall it, use Tools>Extension Manager in Visual Studio.

You can work with models that contain links on computers where the extension is not installed. The links should not be lost.
